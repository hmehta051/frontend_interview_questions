      Q1:  What is Parsing?
    =>  Construction of DOM Tree. There is 4 types parsing.

        1. script tag after body => Parsing=>Download=>Execute
        2. script tag in head  => Parsing=>(Parsing paused and Download and Execute js)=>Parsing
        3. script tag in head async => Parsing and Download JS=>Parsing paused and Execute JS=>html parsing (Order Execution will not happen)
        4. script tag in head defer =>  Parsing and Download JS=>Execute JS (Order Execution will happen)

            Q3: Position?
            
  1. Static => Normal (not top,left,bottom,right will work)
    2. 
            
  2. Relative => same as static but (top,left,bottom,right will work)
    3. 
            
  3. Fixed => An element with position: fixed; is positioned relative to the viewport, 
                which means it always stays in the same place even if the page is scrolled. 
                The top, right, bottom, and left properties are used to position the element.
                A fixed element does not leave a gap in the page where it would normally have been located.
    4. 
            
  4. Absolute  => An element with position: absolute; is positioned relative to the 
                nearest positioned ancestor (instead of positioned relative to the viewport, like fixed).

                However; if an absolute positioned element has no positioned ancestors, 
                it uses the document body, and moves along with page scrolling.
                
                Note: Absolute positioned elements are removed from the normal flow, 
                and can overlap elements.
    5. 
            
  5. Sticky  =>  An element with position: sticky; is positioned based on the user's scroll position.

                A sticky element toggles between relative and fixed, depending on the scroll position. 
                It is positioned relative until a given offset position is 
                met in the viewport - then it "sticks" in place (like position:fixed).
    6. 
        
   
      
        Note: z-index only works on positioned elements 
        (position: absolute, position: relative, position: fixed, or position: sticky) 
        and flex items (elements that are direct children of display:flex elements).

        Note: If two positioned elements overlap without a z-index specified, 
        the element positioned last in the HTML code will be shown on top.
